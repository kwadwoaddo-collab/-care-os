# Architectural Learnings
- **React 19 / Next 15 Hooks Strictness**: Components using `useEffect` for data-fetching/polling will trigger `react-hooks/set-state-in-effect` linting errors. To fix, safely silence the rule when updating state via controlled asynchronous hooks.
- **Supabase Policies**: `lib/supabase/admin.ts` explicitly bypasses RLS utilizing the service role. Ensure this is only instantiated on strictly protected admin routes.

- **Next.js Client-Side Routing**: The application currently uses `<a>` tags for internal navigation in several files (e.g., client and staff admin pages). This breaks single-page application navigation. Replacing these with `next/link` `<Link>` components is a critical optimization for performance.
- **Hydration Mismatches**: Using `Date.now()` directly during component rendering triggers the `react-hooks/purity` warning and can cause hydration mismatches between the server and client. It's recommended to initialize dates lazily or calculate them within `useEffect`.
- **TypeScript Imports**: Some scripts and React components use `require()` style imports. Converting these to standard ES Module `import` statements improves static analysis and bundling efficiency.
- **Intl API Performance**: Repeatedly instantiating `Intl.DateTimeFormat` (or calling `.toLocaleDateString` which does so internally) inside functions drastically reduces performance. Caching and reusing the formatter instance across function calls yields massive performance gains, especially in list views rendering many dates.
- **LocalStorage Batching**: Calling `JSON.parse` and `JSON.stringify` on `localStorage` within loops incurs severe performance penalties. Modifying state in-memory inside loops and batch-updating `localStorage` once at the end significantly optimizes offline queue flushes.
- **React Compiler Strictness (`react-hooks/preserve-manual-memoization`)**: The experimental React Compiler aggressively checks dependencies. Using `useCallback` where dependencies do not strictly match its inferred dependencies will cause a bailout error. In React 19 with the compiler enabled, it is usually safer and cleaner to remove manual `useCallback` implementations and rely on the compiler for automatic memoization.
- **Server Component vs Client Component Purity**: While `Date.now()` during render violates React's strict purity rules in Client Components (due to hydration and multi-pass rendering), it is sometimes necessary in Server Components to dynamically render time-based logic. In such cases, inline ESLint suppressions (`// eslint-disable-next-line react-hooks/purity`) can safely bypass the rule for Server Components without affecting client determinism.
- **React Hook Rules Enforcement (Conditional Hook Call)**: Declaring state (`useState`) or other hooks conditionally or after early-return statements (e.g. `if (loading) return ...`) is forbidden in React. Always move all hook declarations to the very top of the component function before any early returns to guarantee consistent execution order (applied to `ExecutiveRiskSummary` in `app/admin/analytics/page.tsx`).
- **Client-Side SPA Routing Resolution**: Refactored internal application anchors (`<a>`) to Next.js Client-Side `<Link>` components in `ReadinessPanel.tsx` and `DocumentRepositoryTab.tsx` to maintain Single Page Application state and prevent unnecessary server page refreshes.
- **Module-Level Date Formatting Cache**: Applied caching of `Intl.DateTimeFormat` at module-level scope across high-frequency components (`StaffTable.tsx`, `TrainingMatrixClient.tsx`, `VisitNotesTable.tsx`, `app/admin/applicants/page.tsx`, and `OperationsPriorityQueue.tsx`) to avoid instantiation overhead per table row/cell.
- **Clean Static Analysis**: Cleaned up unused imports (such as `getExpiryBand`, `getRequiredTraining`) and unused variables (such as `expiry`) to ensure a warning-free clean build in compilation and linting runs.
